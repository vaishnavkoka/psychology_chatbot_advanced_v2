## Question 2: Chrome Instances - How Many Will Open?

**Your Question:** "Based on our features, how many Chrome instances will get opened?"

**Answer:** **ONLY 1 Chrome Instance for ALL 21 Test Executions** ✅

### Breakdown:

```
Total Scenarios: 20 (with 21 total test runs due to Scenario Outline expansion)

Without Driver Reuse:
Scenario 1: Open Chrome #1, Run, Close → Total: 1 browser
Scenario 2: Open Chrome #2, Run, Close → Total: 2 browsers
Scenario 3: Open Chrome #3, Run, Close → Total: 3 browsers
...
Scenario 21: Open Chrome #21, Run, Close → Total: 21 browsers
Result: 21 Chrome instances opened! ❌ VERY SLOW

WITH Driver Reuse (Singleton Pattern - WHAT WE DID):
Scenario 1: Open Chrome #1 (DriverManager.getDriver()) → 1 browser
Scenario 2: REUSE Chrome #1 (driver already exists) → 1 browser
Scenario 3: REUSE Chrome #1 → 1 browser
...
Scenario 21: REUSE Chrome #1 → 1 browser
After All: Close Chrome #1 (DriverManager.closeDriver())
Result: 1 Chrome instance for ALL 21 tests! ✅ 80% FASTER
```

### How It Works:

**DriverManager.java (Singleton Pattern):**
```java
public class DriverManager {
    private static WebDriver driver;  // One instance, static
    
    public static WebDriver getDriver() {
        if (driver == null) {
            // First scenario: Create browser
            initializeDriver();
        }
        // All other scenarios: Return existing browser
        return driver;
    }
}
```

**StepDefinitions.java:**
```java
public class StepDefinitions {
    private WebDriver driver;
    
    public StepDefinitions() {
        // Every step class gets the SAME driver instance
        this.driver = DriverManager.getDriver();
    }
}
```

**Timeline:**
- Scenario 1 executes → Chrome opens (1st getDriver() call)
- Scenario 2 executes → REUSES same Chrome (2nd getDriver() call returns existing)
- Scenario 3-21 execute → All REUSE same Chrome
- After all tests → Chrome closes (DriverManager.closeDriver())

### Performance Impact:
- **Without Reuse:** 21 Chrome launches + 21 Chrome closes = VERY SLOW
- **With Reuse:** 1 Chrome launch + 1 Chrome close = 80% FASTER ⚡

### 2. ONE Chrome Instance (Singleton Pattern)
```java
public class DriverManager {
    private static WebDriver driver;  // Single static instance
    
    public static WebDriver getDriver() {
        if (driver == null) {
            // First call: Create browser
            initializeDriver();
        }
        // All other calls: Return existing browser
        return driver;
    }
    
    public static void closeDriver() {
        if (driver != null) {
            driver.quit();  // Close after all tests
            driver = null;
        }
    }
}

// Result:
// Scenario 1: driver == null → Creates Chrome
// Scenario 2-21: driver != null → REUSES Chrome
// After All: closeDriver() → Closes Chrome
// Total: 1 Chrome instance!
```
